# 컴퓨터과학 기초

CS(Computer Science) 공부의 목적은 면접 대비가 아니라 시스템이 왜 그렇게 동작하는지 설명하는 능력입니다.

## 자료구조와 알고리즘

우선순위:

- 배열, 연결 리스트, 스택, 큐, 해시 테이블, 트리, 힙, 그래프
- 정렬, 이분 탐색, BFS, DFS, 다익스트라, 동적 계획법
- 시간 복잡도와 공간 복잡도

실무 연결:

- React 렌더링에서 리스트 key가 왜 중요한지 설명합니다.
- PostgreSQL 인덱스가 B-Tree 기반으로 동작하는 이유를 이해합니다.
- 캐시와 해시 충돌의 비용을 설명합니다.

## 운영체제

우선순위:

- 프로세스와 스레드
- 컨텍스트 스위칭
- 메모리, 가상 메모리, 페이지
- 파일 시스템
- 동시성과 잠금

실무 연결:

- Spring 서버의 스레드 풀이 왜 고갈되는지 설명합니다.
- DB(Database) 커넥션 풀과 애플리케이션 스레드 풀의 관계를 이해합니다.
- 동시 요청에서 race condition을 재현합니다.

## 네트워크

우선순위:

- TCP/IP, DNS, TLS
- HTTP(HyperText Transfer Protocol) 요청과 응답
- 캐시 헤더
- 로드밸런싱
- 타임아웃과 재시도

실무 연결:

- API(Application Programming Interface)가 느릴 때 브라우저, 서버, DB 중 병목을 좁힙니다.
- 재시도가 중복 결제를 만들 수 있는 이유를 설명합니다.
- CORS를 보안 모델 관점에서 이해합니다.

## 데이터베이스

우선순위:

- 관계형 모델
- 정규화와 반정규화
- 인덱스
- 트랜잭션과 격리 수준
- 잠금과 교착 상태

실무 연결:

- JPA(Java Persistence API)의 N+1 문제를 SQL(Structured Query Language) 실행 관점에서 설명합니다.
- 느린 API의 원인이 쿼리인지 애플리케이션인지 분리합니다.
- 데이터 무결성을 애플리케이션 코드에만 맡기지 않습니다.

## 체크리스트

- [ ] 개념을 그림 없이 말로 설명할 수 있다.
- [ ] 개념을 Spring, React, PostgreSQL 사례와 연결할 수 있다.
- [ ] 성능 문제를 측정 없이 추측으로만 판단하지 않는다.
- [ ] 알고리즘 문제 풀이와 실무 병목 분석을 함께 훈련한다.
