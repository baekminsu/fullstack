# 커리어 성장 루틴

20년차 개발자의 차이는 지식량보다 판단의 품질에서 드러납니다. 판단은 기록, 피드백, 반복으로 좋아집니다.

## 분기별 목표

매 분기 하나의 큰 주제를 정합니다.

- Q1: Spring 백엔드 깊게 파기
- Q2: React와 사용자 경험
- Q3: PostgreSQL과 데이터 모델링
- Q4: 아키텍처, 테스트, 운영

주제는 반복해도 됩니다. 같은 주제를 더 깊은 수준에서 다시 보는 것이 장기 성장에 유리합니다.

## 리뷰 습관

코드 리뷰에서 볼 것:

- 요구사항을 정확히 만족하는가?
- 실패 케이스와 예외 처리가 있는가?
- 테스트가 위험한 변경을 보호하는가?
- 이름과 경계가 미래 변경을 쉽게 만드는가?
- 성능, 보안, 데이터 무결성 위험은 없는가?

설계 리뷰에서 볼 것:

- 문제 정의가 명확한가?
- 대안이 비교되었는가?
- 의존성과 데이터 흐름이 설명되는가?
- 운영 중 실패했을 때 추적할 수 있는가?
- 지금 만들지 않아도 되는 복잡성이 들어갔는가?

## 실력 측정 방법

초급에서 중급:

- 주어진 요구사항을 안정적으로 구현합니다.
- 기본 테스트와 문서를 남깁니다.
- 프레임워크 사용법을 찾아 적용할 수 있습니다.

중급에서 시니어:

- 요구사항의 모호함과 위험을 먼저 발견합니다.
- 변경 가능성을 기준으로 구조를 잡습니다.
- 장애와 운영 비용을 고려합니다.

시니어에서 리드:

- 팀의 반복 문제를 줄이는 시스템을 만듭니다.
- 기술 선택을 비즈니스, 조직, 운영 맥락과 연결합니다.
- 다른 사람이 더 좋은 결정을 내리도록 돕습니다.
