# React 프론트엔드 학습 가이드

React 학습의 목표는 컴포넌트를 많이 아는 것이 아니라 사용자 흐름을 안정적으로 표현하는 것입니다.

## 학습 순서

1. TypeScript 기본기
2. JSX와 컴포넌트
3. props와 state
4. 이벤트와 폼
5. 서버 상태와 비동기 처리
6. 라우팅
7. 접근성과 키보드 사용성
8. 성능 최적화
9. 테스트

## 컴포넌트 설계 기준

- 페이지 컴포넌트는 사용자 흐름을 조립합니다.
- 도메인 컴포넌트는 업무 의미를 드러냅니다.
- 공용 UI 컴포넌트는 시각적 재사용을 담당합니다.
- 커스텀 훅은 상태 흐름과 부수 효과를 분리합니다.
- API(Application Programming Interface) 클라이언트는 화면 코드에서 직접 흩어지지 않게 합니다.

## 상태 분류

- Local UI State: 모달 열림, 입력 중인 값, 탭 선택
- Server State: API에서 가져온 데이터, 로딩, 에러, 캐시
- URL State: 검색 조건, 페이지 번호, 정렬
- Global Client State: 로그인 사용자, 테마처럼 앱 전역에서 필요한 값

상태를 잘못 분류하면 컴포넌트가 비대해지고 버그가 늘어납니다.

## 프로젝트 과제

1. 로그인 화면과 인증 유지
2. 게시글 목록, 상세, 작성, 수정
3. 검색, 필터, 페이지네이션
4. 에러 화면과 빈 상태
5. 폼 검증과 서버 검증 메시지 표시
6. 관리자 대시보드
7. 접근성 점검
8. 렌더링 성능 측정

## 체크리스트

- [ ] 로딩, 에러, 빈 상태가 빠지지 않는다.
- [ ] 폼 입력과 서버 검증 실패가 자연스럽게 연결된다.
- [ ] URL로 현재 화면 상태를 공유할 수 있다.
- [ ] 컴포넌트가 데이터 요청 세부사항에 과하게 묶이지 않는다.
- [ ] 키보드만으로 주요 흐름을 사용할 수 있다.
