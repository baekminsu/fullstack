# 풀스택 프로젝트 포트폴리오

좋은 포트폴리오는 기능 목록보다 문제 해결 과정과 기술 판단을 보여줍니다.

## 프로젝트 1: 개인 지식 관리 서비스

목표:

- Markdown 노트 작성
- 태그와 검색
- 공개 및 비공개 설정
- 버전 기록

집중 역량:

- Spring CRUD와 인증
- React 폼과 라우팅
- PostgreSQL 검색과 인덱스
- Clean Architecture의 가벼운 적용

## 프로젝트 2: 팀 협업 이슈 트래커

목표:

- 프로젝트, 이슈, 댓글, 담당자
- 상태 워크플로우
- 알림
- 권한

집중 역량:

- 도메인 모델링
- 권한 설계
- 복잡한 쿼리
- 실시간 또는 비동기 알림

## 프로젝트 3: 커머스 주문 시스템

목표:

- 상품, 장바구니, 주문, 결제 흐름
- 쿠폰과 포인트
- 재고 차감
- 주문 취소와 환불

집중 역량:

- 트랜잭션
- 동시성
- 장애 처리
- 중복 요청 방지

## 프로젝트 4: 운영 가능한 SaaS 대시보드

목표:

- 조직과 사용자
- 역할 기반 권한
- 통계 대시보드
- 감사 로그
- 사용량 제한

집중 역량:

- 멀티테넌시 기초
- 대시보드 UX(User Experience)
- 집계 쿼리
- 관측 가능성

## 각 프로젝트의 필수 문서

- 문제 정의
- 요구사항
- API(Application Programming Interface) 명세
- ERD
- 아키텍처 결정 기록
- 테스트 전략
- 배포 방법
- 장애 시나리오
- 회고
