# JavaScript + React 프로젝트 커리큘럼

React 학습은 작은 화면을 많이 만드는 것보다 하나의 사용자 흐름을 끝까지 완성하는 것이 중요합니다.

## 1단계: JavaScript 미니 과제

주제:

- Todo 데이터 필터링
- 쇼핑 장바구니 총액 계산
- 사용자 목록 검색
- 비동기 API(Application Programming Interface) 호출 연습

필수 적용:

- `map`, `filter`, `reduce`
- 객체와 배열 불변 업데이트
- Promise
- async와 await
- 에러 처리

완료 기준:

- 원본 데이터를 직접 변경하지 않습니다.
- 실패 케이스를 처리합니다.
- 함수 이름이 계산 의도를 드러냅니다.

## 2단계: 브라우저 DOM(Document Object Model) 과제

주제:

- 검색 가능한 연락처 목록
- 간단한 메모장
- 탭 UI(User Interface)

필수 적용:

- DOM(Document Object Model) 선택
- 이벤트 처리
- form 제출
- localStorage
- URLSearchParams

완료 기준:

- 새로고침 후 필요한 상태가 유지됩니다.
- 이벤트 전파 문제를 설명할 수 있습니다.
- HTML 기본 동작을 이해하고 제어합니다.

## 3단계: React 기본 앱

주제:

- 게시글 목록과 상세

필수 적용:

- JSX(JavaScript XML)
- 컴포넌트
- props
- state
- 조건부 렌더링
- 목록과 key

완료 기준:

- 로딩, 에러, 빈 상태가 있습니다.
- 컴포넌트 이름이 역할을 드러냅니다.
- key로 인덱스를 남용하지 않습니다.

## 4단계: API(Application Programming Interface) 연동 앱

주제:

- Spring API와 연결한 게시판

필수 적용:

- fetch
- API 클라이언트 분리
- 라우팅
- query string
- form 검증
- 서버 검증 메시지 표시

완료 기준:

- URL로 검색 결과를 공유할 수 있습니다.
- API 실패를 사용자에게 보여줍니다.
- 요청과 응답 타입을 문서화합니다.

## 5단계: 인증 앱

주제:

- 로그인, 내 정보, 관리자 화면

필수 적용:

- 인증 상태 모델링
- 보호 라우트
- 권한별 메뉴
- 토큰 만료 처리
- 접근 거부 화면

완료 기준:

- 인증 확인 중과 미로그인을 구분합니다.
- 권한 없는 사용자의 접근을 막습니다.
- 새로고침 후 인증 상태를 복구합니다.

## 6단계: 품질 개선

주제:

- 기존 앱 성능, 테스트, 접근성 보강

필수 적용:

- 컴포넌트 테스트
- 사용자 행동 테스트
- 접근성 점검
- 번들 분석
- 렌더링 병목 측정

완료 기준:

- 핵심 사용자 흐름에 테스트가 있습니다.
- 키보드만으로 사용할 수 있습니다.
- 성능 개선 전후 측정 결과가 있습니다.
