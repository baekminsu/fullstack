# 학습 시스템

장기 성장은 의지보다 시스템에 더 많이 좌우됩니다. 이 문서는 매주 반복할 학습 루틴을 정의합니다.

## 주간 루틴

월요일:

- 이번 주에 만들 기능 하나를 정합니다.
- 기능의 성공 기준, API, DB 변경, UI 흐름을 짧게 씁니다.

화요일부터 목요일:

- 매일 90분 이상 구현합니다.
- 모르는 개념은 즉시 깊게 파기보다 기록만 하고 구현 흐름을 유지합니다.
- 테스트를 먼저 추가하거나, 최소한 수정 직후 테스트를 남깁니다.

금요일:

- 느린 쿼리, 복잡한 컴포넌트, 어려웠던 설계 결정을 하나 고릅니다.
- 원인과 대안을 문서로 정리합니다.

주말:

- 책이나 공식 문서를 읽고 요약합니다.
- 이번 주 결과물을 리팩터링하거나 회고합니다.

## 하루 학습 단위

1. 목표를 한 문장으로 씁니다.
2. 25분 동안 현재 이해를 바탕으로 직접 시도합니다.
3. 막힌 지점을 구체적으로 기록합니다.
4. 문서, 책, 코드 예제로 확인합니다.
5. 다시 구현하고 테스트합니다.
6. 배운 내용을 5줄 이상으로 정리합니다.

## 기록 규칙

- 배운 개념은 "정의, 필요한 이유, 예시, 실수하기 쉬운 점"으로 정리합니다.
- 프로젝트 기록은 "요구사항, 설계 선택, 구현, 테스트, 회고"로 정리합니다.
- 회고는 감상보다 다음 행동이 나오게 씁니다.

## 월간 점검

매월 마지막 주에 아래 질문에 답합니다.

- 이번 달에 실제로 만든 기능은 무엇인가?
- 테스트가 보호해준 버그가 있었는가?
- 설계 때문에 쉬웠던 변경과 어려웠던 변경은 무엇인가?
- PostgreSQL에서 새로 이해한 실행 계획이나 잠금 사례가 있는가?
- React에서 사용자 경험이나 성능을 개선한 사례가 있는가?
- 다음 달에 줄일 나쁜 습관은 무엇인가?
